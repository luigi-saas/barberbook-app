# i18n Rules (apps/web)

- All user-facing text in `apps/web` must use `useTranslations()` from next-intl — never hardcode strings
- When adding new text, always add the key to ALL locale files: en, fr, es, de, pt, zh, ar
- The `[locale]` segment is mandatory in all routes under `apps/web/app/[locale]/`
- The middleware at `apps/web/middleware.ts` handles locale detection and redirects — do not bypass it
- The `NextIntlProvider` in `apps/web/app/[locale]/NextIntlProvider.tsx` wraps all locale-aware layouts
- Use `getTranslations()` (async) for Server Components and `useTranslations()` for Client Components
- Locale config lives in `apps/web/next-intl.config.js` — update it when adding new locales
